## Session 2 — snapshot-vs-baseline precedence fix (#151, verified-row pinning)
User: "First the snapshot vs baseline." Traced the runtime pricing path:
buildSpendEngine (bootstrap.go) returns the EFFECTIVE engine (flat baseline is
only a fallback). SnapshotsToDatedTables layers each snapshot as
DefaultTable().MergeOverrides(snapshot), effective-dated by FetchedAt → the
LATEST snapshot wins for current events. CONFIRMED live: despite #149 correcting
deepseek-chat to $0.14/$0.28, the adopted LiteLLM snapshot still priced it
$0.28/$0.42 (LiteLLM leaves the deprecated alias stale); same for mistral-small
(LiteLLM Small 3.2 $0.06/$0.18 vs vendor Small 4 $0.15/$0.60). The Opus lesson,
live: an automated source silently overriding human verification — and the
consistency guard is Anthropic-only, so no protection on these.

Also found: snapshots are dated by FetchedAt, NOT price-effective-from — so a
stale source rate gets stamped "effective <fetch date>", a small lie.

FIX (user chose "pin verified rows" over "baseline-authoritative" / "document"):
`verified: true` marker on a catalog row → spend.DefaultPinnedKeys() → 
SnapshotsToDatedTables strips pinned keys from each snapshot's override table
(base.MergeOverrides(snap.Table().Without(pinned))) so baseline wins at every
period. Unpinned rows still auto-adopt; new models still surface. Pinned the
non-Anthropic vendor-verified rows (mistral large/medium/small/codestral,
deepseek chat/reasoner, o1, grok-3). Verify-then-pin = the hardening workflow.
Test: TestEffectiveEngine_VerifiedRowResistsStaleSnapshot (stale deepseek snapshot
loses to pinned baseline; unpinned gpt-4o still adopts). Admin-merged (#151).

GOTCHA: pricing show/diff render the RAW snapshot (provenance), so pinned rows
DISPLAY the stale source value there and diff lists them as drift — even though

pin status so diff doesn't tempt a human to "fix" the baseline toward the stale
