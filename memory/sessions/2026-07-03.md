## Decisions Made
- Critical-line survival enforced in the ENGINE (`enforceCritical`), not per-formatter — so user config formatters are as safe as built-ins. Chosen because it lets rules be data (config) without losing the guarantee; the invariant lives in code, not in each rule.
- Learning is offline/advisory/gated. `fmt learn --apply` writes only loss-level overrides (safe — never touches critical rules); new-formatter candidates are printed, not auto-written. Reasoning: any runtime self-modification would break the determinism + survival guarantee.
- Cloud CLIs (aws/gcloud/az) pass JSON through untouched — generic consecutive-dup scrub corrupts JSON arrays. Only compress table/text.
- Config formatters excluded from proxy content-sniff (no reliable content signature).
- Skipped low-value RTK-parity commands (ls/cat/find/grep/diff/wget) — mostly signal, generic scrub covers them.
- Released via minor bumps for features (0.26/0.27/0.28), patch for docs (0.28.1).

## Session 5

### Decisions Made
- Removed the caveman plugin entirely (settings.json enabledPlugins + marketplaces + installed_plugins.json + cached dirs). Reasoning grounded in the user's real data: caveman compresses assistant output prose, which is <1% of their token spend (input:output ≈ 807:1; output 0.12% of tokens, ~5% cost-weighted). Removing it cost effectively nothing.
- Built the self-wiring analyzer (internal/infra/jsonlfmt) to answer "where do my tokens go" from the Claude Code logs — no daemon, no wrapped commands. Surfaced honestly that fmt's catalog only touches ~1-4% of the user's Bash (their mix is echo/grep/sed/cat/find + Read). Fixed two real bugs it exposed: depth-1 glob (saw 8% of logs) and cd-attribution (cd && go misattributed to cd).
- Read is the real lever: 46% of context, ~16% re-reads, ~24% dup content, 54% already ranged. Not a formatter problem — context management.
- Built `tokenops read-guard`: a Claude Code PreToolUse hook that blocks redundant unchanged full re-reads at the source (works for Max/OAuth clients that can't route through the proxy). observe (log) vs active (deny). Confirmed the PreToolUse contract via claude-code-guide (can only allow/deny, not modify input → block-or-allow; deny via hookSpecificOutput.permissionDecision).
- Released v0.30.0. Installed read-guard in OBSERVE mode into ~/.claude/settings.json (backup: settings.json.pre-readguard.bak). User wants to watch `read-guard stats` over a few sessions before flipping to active.

- Shipped v0.30.1 after discovering observe mode showed a bare "0 reclaimable" without explaining why. Added repeat/ranged/changed classification to the ledger + stats so it splits repeat reads into reclaimable (unchanged full) vs post-edit (changed — not waste) vs ranged (intentional). Observe is now self-explaining. Reasoning: the point of observe is to understand re-read patterns; a bare 0 looked like nothing happened.
- HONEST FINDING (the important one): live data across the user's reconnected CC windows shows 7 repeat reads, ALL ranged → 0 reclaimable. The blockable kind (unchanged FULL re-read) is near zero because the user/agent already re-reads via ranges (consistent with 54% ranged overall). So read-guard likely has little/nothing to reclaim for this user — and correctly reports that instead of inventing savings. Recommendation: probably NOT worth flipping to active; keep observing.

Resolved by shipping v0.30.1 + brew upgrade; new events carry the fields. Lesson: when a hook binary's log schema changes, old ledger rows won't backfill — the metric transitions forward only.

## Session 7

### Decisions Made
- **Full provider coverage delivered as OpenAI-compatible framework, not per-provider files** — most new providers (groq/deepseek/xai/perplexity/fireworks/cerebras/together/openrouter) are OpenAI-wire-format, so the keystone was a shared `NewOpenAICompatible(id, prefix, url)` constructor + parameterized `normalizeOpenAICompatible(id)`. Each new provider is now ~1 line + pricing YAML + tokenizer registration. Providers went 4 → 13.
- **Each provider meters with its OWN id, never a shared "openai"** — the normalizer is parameterized by id so groq/etc. attribution is correct. Guarded by a test.
- **Cohere added as first-class non-OpenAI provider** (dedicated `normalizeCohere` for v2 `/v2/chat` + v1 `/v1/chat`), because its Bearer auth works with the passthrough proxy. Bedrock deferred (needs SigV4 → no per-provider auth hook exists).
- **opencode passive reader built on SQLite, not JSONL** — opencode stores sessions in `~/.local/share/opencode/opencode.db` (verified by inspecting the real machine). Opened `mode=ro`; parses `message.data` JSON; multi-provider; uses existing `modernc.org/sqlite` dep. First SQLite-backed vendor reader.
- **Gemini CLI passive reader NOT built** — its `~/.gemini/tmp/<hash>/logs.json` is a prompt log with ZERO token data (verified on real files). Proxy plane only. Honest boundary documented rather than shipping a broken reader.
- **`provider set` hardened** — validates name against known providers (unknown name used to hard-crash the daemon at boot) + supports preset URLs (omit URL). Fixes a real UX landmine the mapper found.
- **Cut v0.33.0** — 13 providers + opencode + coverage docs = clean minor bump.
- **Blog setup/use-cases section** — added "Point it at your own usage" (4 personas, escalating commitment) to the 800:1 post with verified commands.
