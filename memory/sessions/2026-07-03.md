## Decisions Made
- Critical-line survival enforced in the ENGINE (`enforceCritical`), not per-formatter — so user config formatters are as safe as built-ins. Chosen because it lets rules be data (config) without losing the guarantee; the invariant lives in code, not in each rule.
- Learning is offline/advisory/gated. `fmt learn --apply` writes only loss-level overrides (safe — never touches critical rules); new-formatter candidates are printed, not auto-written. Reasoning: any runtime self-modification would break the determinism + survival guarantee.
- Cloud CLIs (aws/gcloud/az) pass JSON through untouched — generic consecutive-dup scrub corrupts JSON arrays. Only compress table/text.
- Config formatters excluded from proxy content-sniff (no reliable content signature).
- Skipped low-value RTK-parity commands (ls/cat/find/grep/diff/wget) — mostly signal, generic scrub covers them.
- Released via minor bumps for features (0.26/0.27/0.28), patch for docs (0.28.1).
